php5.6からphp7へ移行した

2016年01月01日 14時40分

年も明けたのでこのサーバ（CentOS7）のphpを5.6から7に移行しました。

評判通り体感できるレベルで高速化された。
バージョンあげるだけで速度が改善されるなんてありがたい。

その際の作業メモ。

インストール手順

# php5.6のアンインストール
sudo yum remove php*
# remi リポジトリは追加済み
sudo yum install --enablerepo=epel,remi,remi-php70 php php-cli php-devel php-mbstring php-pecl php-xml php-json php-imagick
# http server 再起動
systemctl restart httpd

Laravel4製のアンテナサイトは全くのノントラブル、piwik、mediawikiも問題なし。
ただWordpressが画面真っ白になってしまったので調査開始。

トラブル.2 プラグインがphp7対応できていない

ログを確認してみるとplugins/以下のファイルでsplitとかeregiなどが使われていてエラーになっていた。
消しても問題無さそうなプラグインだったので削除して回避。

発生したトラブルはこれだけだった。
本体は全く問題なく動いている。

7.0移行用のチェッカーを使うとソース側の問題は先に潰すことができる。

PHP 7 Compatibility Checker(php7cc)
https://github.com/sstalle/php7cc

(function(){
var t = document.querySelectorAll("*"),
k, u;
for(k in t){
u = document.createElement('span');
u.textContent = "?";
t.item(k).appendChild(u);
}
})();

「第1回FlectMeetup MEANスタック＋HerokuでWebサービスを作ろう！」に参加してきた

仕事でAngularJSと絡む機会が出てきそうだったので「第1回FlectMeetup MEANスタック＋HerokuでWebサービスを作ろう！」に参加してきました。
https://flect.doorkeeper.jp/events/29804

今回の勉強会はハンズオンだったのですが、環境構築で躓く方々が続出で資料を見ながらの半モクモク会のような感じでした。
私は特にトラブル無く目標通りに進められました。

その充実した資料はこちらです。
https://github.com/FLECT-DEV-TEAM/mean-heroku-tutorial

MEAN、herokuほぼ全てまともに触ったことが無かったので全てが新鮮でした。
heroku凄い！これは使わない手はない！と今更ですが感動です。
やっぱり勉強会は知識のレンジを広げるのにとても有用です。触れる機会を作れたので後は継続的に使っていって理解を深めていきたいと思います。

正直、今から1.xと互換性がない2.x が来年でるとわかっている状態で1.x系へのモチベーションがなかなか湧かなかったのですがそれが払拭できました。

主催するFLECTさんの初勉強会だったらしいのですが、会場案内や進行などもスムーズで懇親会までとても楽しく参加させていただきました。ありがとうございました。

FLECTさんは新しい技術に積極的に取り組める、積極的なエンジニアを募集しています。

新築ビルに引越すという会社としての勢い、スタッフの方々から溢れ出る技術力を感じる会社だったので転職を考えている方は一度HPを見てみては如何でしょうか。
